### Context
n8n을 활용하여 디스코드 메시지를 트리거로 에이전트(Agent)를 실행하고, 이를 TIL(Today I Learned) 형식으로 변환하여 GitHub 저장소에 자동 Push 및 README.md 업데이트까지 수행하는 자동화 워크플로우를 구축했습니다. n8n의 노드 기반 아키텍처를 이해하고 데이터 흐름을 설계하는 과정을 기록합니다.

### Core
워크플로우는 다음과 같은 핵심 단계로 구성됩니다.

* **Trigger**: Discord Trigger 노드를 통해 특정 채널의 메시지를 실시간으로 수신합니다.
* **Agent**: LLM 노드(또는 AI Agent 노드)를 활용하여 수신된 메시지를 TIL 형식의 Markdown 파일 구조로 가공합니다.
* **GitHub Integration**: GitHub 노드를 사용하여 가공된 콘텐츠를 새로운 파일로 저장소에 Commit 및 Push합니다.
* **README Update**: 기존 README.md 파일을 불러온 후(Read), 새로 생성된 TIL 인덱스를 추가하여 다시 Push(Update)합니다.
* **Notification**: 최종 작업 완료 상태를 Discord 채널로 다시 전송합니다.

```json
// n8n 워크플로우 구조 예시 (데이터 흐름 개념)
{
  "trigger": "discord_message_received",
  "processing": "llm_text_transformation",
  "actions": [
    "github_create_file",
    "github_update_readme",
    "discord_send_confirmation"
  ]
}
```

### Insight
n8n 자동화 설계 시 가장 중요한 핵심 원칙은 각 노드의 **입력 데이터 구조(Input)**와 **출력 데이터 결과(Output)**를 정확하게 정의하는 것입니다. 모든 노드는 이전 단계로부터 전달받은 JSON 데이터를 기반으로 동작하며, 이 데이터의 필드명과 구조를 명확히 이해하고 있다면 복잡한 로직도 쉽게 모듈화할 수 있습니다. 노드 간의 데이터 흐름(Data Mapping)만 제대로 파악한다면 자동화 구현의 난이도는 크게 낮아집니다.
